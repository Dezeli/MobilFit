🐛 버그 수정

  1. 회원가입 시 비밀번호 눈 아이콘이 반대로 동작하던 문제
  "비밀번호" 입력칸의 눈 아이콘을 눌렀는데 정작 "비밀번호 확인" 칸이 보이고, 반대도
  마찬가지였습니다. 두 필드가 서로의 토글을 보고 있던 코드를 잡아서 이제 각자 자기 칸의
  표시/숨김만 제어합니다.

  2. 회원 탈퇴 직후 화면이 어색하게 넘어가던 문제
  "회원 탈퇴 완료" 알럿이 떠 있는 동시에 로그인 화면으로 즉시 이동해서, 알럿이 로그인 화면
  위에 둥둥 떠 있는 모양이 됐었습니다. 이제는 사용자가 "확인"을 눌러야 로그인 화면으로
  이동합니다.

  3. 로그인/회원가입 로딩 스피너가 안 돌던 문제
  로딩 중 표시되는 동그란 스피너가 사실은 그냥 멈춰 있는 정적인 도형이었습니다. 진짜로 360도
  회전하도록 애니메이션을 붙여서, 로그인 / 인증코드 전송 / 인증코드 확인 / 회원가입 버튼
  모두에서 실제로 돌아갑니다.

  ✏️ UX 개선

  주행 기록 안내 문구가 실제와 안 맞던 문제
  주행 기록 페이지에 "총 N회의 주행 기록이 있습니다"라고 적혀 있었는데, 실제 API는 최근
  2주치만 내려주고 있었습니다. 사용자가 "내 평생 기록이 N회뿐이구나" 오해할 수 있어서 → "최근
   2주간 N회의 주행 기록이 있습니다"로 바로잡았습니다.

  🧹 내부 정리 (사용자에게는 안 보임)

  중복 코드 제거
  - 이메일 형식 검증 함수가 로그인/아이디찾기/비번재설정/회원가입 4곳에 똑같이 복붙돼 있었고,
   시간 포맷팅 함수도 랭킹/주행기록 2곳에 중복돼 있었습니다. → 공통 유틸 파일(lib/utils.ts)로
   한 군데 모았습니다. 앞으로 검증 규칙을 바꿀 때 한 군데만 고치면 됩니다.

  Expo 템플릿 잔재 청소
  - 프로젝트 초기에 Expo가 자동 생성해줬지만 실제로는 안 쓰는 컴포넌트 5개(Collapsible,
  ExternalLink, HelloWave, ParallaxScrollView, IconSymbol.ios) 삭제.
  - 코드 어디서도 import하지 않는 스타일 정의들(요금 차트 스타일, 환경 비주얼 스타일, 푸터
  스타일, 랜딩 페이지의 반원 쇼케이스 스타일 등) 약 400줄 정리.

  의존성 다이어트
  - 안 쓰는 패키지 6개 제거: @here/harp-utils, @here/olp-sdk-core, @mapbox/polyline, geolib,
  react-native-root-toast, react-native-toast-message. 앱 번들 크기와 설치 시간이 줄어듭니다.
  - eas-cli는 빌드 도구인데 실서비스 의존성으로 잘못 분류돼 있던 걸 개발 의존성으로
  옮겼습니다 (앱 빌드 결과물에 포함되지 않게).
  - expo-dev-client 추가 → 개발 빌드를 더 편하게 만들기 위함.

  📊 규모

  전체적으로 약 600줄 순감소한 정리 위주의 변경입니다. 새 기능은 없고, 깨져 있던 동작 3개를
  고치고 안내 문구 1개를 바로잡았으며, 나머지는 코드를 가볍고 깔끔하게 만드는 작업입니다.

  ⚠️ 커밋 전 권장 확인

  지운 패키지들(특히 geolib, @mapbox/polyline)은 GPS/경로 관련 라이브러리라서, 혹시 어디선가
  import만 살짝 남아있는지 한 번 검색해보고 커밋하시는 걸 추천드립니다. 원하시면 제가 확인해
  드리겠습니다.
